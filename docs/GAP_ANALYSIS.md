# 项目差距分析

对比原 n8n 信息聚合器方案与当前 GitHub Actions 实现的差距。

## 功能对比

| 功能 | n8n 原方案 | 当前实现 | 状态 |
|------|-----------|----------|------|
| RSS 采集 | - | 支持 | 新增 |
| Reddit 采集 | 支持 | 代码已有，未启用 | 待API |
| Twitter 采集 | 支持 | 支持 | 完成 |
| YouTube 采集 | 支持 | 支持 | 完成 |
| AI 翻译 | DeepSeek | Claude（已禁用） | 待修复 |
| AI 摘要 | 60-80字摘要 | 20-30字摘要 | 待优化 |
| 数据存储 | 飞书多维表格 | GitHub Pages HTML | 已替代 |
| 定时触发 | n8n 内置 | GitHub Actions | 完成 |

## 详细差距

### 1. 数据过滤逻辑

**n8n 原方案**：
- Reddit: 24小时内 + 点赞>50
- YouTube: 获取视频统计信息（播放量、点赞、评论数）

**当前实现**：
- YouTube: 仅按播放量排序，未做热度过滤
- 缺少统一的热度过滤机制

### 2. AI 处理差异

**n8n 原方案**：
- 使用 DeepSeek 模型（成本低）
- 详细的摘要提示词（60-80字）
- 结构化输出解析

**当前实现**：
- 使用 Claude API（成本较高）
- 简单的摘要提示词（20-30字）
- API 代理兼容性问题待解决

### 3. 未实现功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 飞书同步 | 低 | 原方案存储到飞书多维表格 |
| 去重机制 | 中 | 避免重复内容 |
| 热度排序 | 中 | 按互动数据排序展示 |
| 历史归档 | 低 | 保留历史报告 |

## 后续优化建议

1. **短期**：修复 AI 处理，尝试更换模型或优化提示词
2. **中期**：添加热度过滤和排序逻辑
3. **长期**：支持更多数据源和输出方式
